\label{app:realign_algorithm_compute}

This appendix keeps the main paper compact while making the implementation easy to audit. Cross-references from the main text point to the specific subsection needed for each result.

\subsection{Training Algorithm and Compute}

LARA adds a lightweight training-time branch to the diffusion transformer. The backbone forward pass exposes source and target activations, the shared predictor maps the source state into target-layer coordinates, and a small depth-shortcut subset can be routed through the predicted state inside the same flow-matching objective. The auxiliary regularizer combines transition prediction, path consistency, and weak-layer diversity.

\begin{algorithm}[!htbp]
\caption{\textbf{LARA training.} Flow matching with an integrated depth shortcut.}
\label{alg:realign_training}
\small
\begin{algorithmic}[1]
\Require backbone $v_\theta$ with prefix/suffix maps $F_{\theta,1:a},F_{\theta,b:L}$; online predictor $G_\psi$ and EMA predictor $G_{\bar\psi}$.
\Require timestep law $p_t$; pair law $q(a,b,c)$; depth-shortcut subset $\mathcal{B}_{\mathrm{ds}}$; weights $\lambda_{\mathrm{pred}},\lambda_{\mathrm{cons}},\lambda_{\mathrm{div}}$.
\Repeat
    \State Sample $x_0\sim p_{\mathrm{data}}$, condition $y\sim p(y\mid x_0)$, noise $\epsilon\sim\mathcal{N}(0,I)$, timestep $t\sim p_t$.
    \State Form $x_t=\alpha_t x_0+\sigma_t\epsilon$ and $u_t=\dot\alpha_t x_0+\dot\sigma_t\epsilon$.
    \State Run the full backbone and cache hidden states:
    \[
    (\hat v,\mathcal{H})\leftarrow v_\theta(x_t,t,y),\qquad
    \mathcal{H}=\{A_\ell^t\}_{\ell=0}^{L}.
    \]
    \State Sample $(a,b,c)\sim q(a,b,c)$ with $a<b<c$, and compute transitions from cached states:
    \[
    \begin{aligned}
    \hat A_{a\to b}^t &= G_\psi(A_a^t;e_a,e_b,e_t),\\
    \hat A_{a\to c}^t &= G_\psi(\mathrm{sg}(A_a^t);e_a,e_c,e_t),\\
    \tilde A_{a\to b\to c}^t
    &= G_{\bar\psi}\!\left(\mathrm{sg}(\hat A_{a\to b}^t);e_b,e_c,e_t\right).
    \end{aligned}
    \]
    \Statex \hspace{\algorithmicindent}\textit{This uses two online predictor forwards and one EMA predictor forward; the inner $a\to b$ state is cached, not recomputed.}
    \State Compute the routed velocity for the depth-shortcut subset:
    \[
    \tilde v=F_{\theta,b:L}(\hat A_{a\to b}^t,t,y).
    \]
    \State Compute the unified flow-matching loss:
    \[
    \mathcal{L}_{\mathrm{FM}}
    =
    \mathbb{E}_{i\in\mathcal{B}}
    \left[
    \|\hat v_i-u_{t,i}\|_2^2
    +
    \mathbb{I}_{\{i\in\mathcal{B}_{\mathrm{ds}}\}}
    \|\tilde v_i-u_{t,i}\|_2^2
    \right].
    \]
    \State Compute the LARA regularizer from the cached states in $\mathcal{H}$:
    \[
    \begin{aligned}
    \mathcal{L}_{\mathrm{LARA}}
    &=
    \lambda_{\mathrm{pred}}d(\hat A_{a\to b}^t,\mathrm{sg}(A_b^t))\\
    &\quad+\lambda_{\mathrm{cons}}d(\hat A_{a\to c}^t,\mathrm{sg}(\tilde A_{a\to b\to c}^t))\\
    &\quad+\lambda_{\mathrm{div}}\mathbb{E}_{i<j}
    \operatorname{cos}^2(A_i^t,A_j^t),
    \qquad A_i^t,A_j^t\in\mathcal{H}.
    \end{aligned}
    \]
    \State Update $\theta,\psi$ with $\mathcal{L}_{\mathrm{FM}}+\mathcal{L}_{\mathrm{LARA}}$.
    \State Update the EMA predictor $\bar\psi\leftarrow \mu\bar\psi+(1-\mu)\psi$.
\Until{training budget is reached}
\end{algorithmic}
\end{algorithm}

The timestep law $p_t$ is a protocol choice rather than part of the transition objective. Main LARA runs use the default training sampler reported in \Cref{tab:appendix_imagenet_protocol}; timestep-sampler alternatives are evaluated separately in \Cref{tab:main_hyperparameter_tuning}.

\paragraph{Predictor-induced transition distance.}
The distance term is evaluated on the transition produced by LARA, not on the two raw hidden states. Given a source--target pair $(a,b)$, the source activation is first passed through the layer-aware predictor so that the comparison is made in the target-layer coordinate system. The target activation is used only as a fixed reference, and the final score averages the tokenwise directional mismatch.

\begin{algorithm}[!htbp]
\caption{\textbf{Predictor-induced transition distance in LARA.}}
\label{alg:lara_transition_distance}
\small
\begin{algorithmic}[1]
\Require source state $A_a^t$; target state $A_b^t$; transition predictor $G_\psi$; conditioning tuple $(e_a,e_b,e_t)$.
\State $\widehat A_{a\to b}^{\,t}\gets G_\psi(A_a^t;e_a,e_b,e_t)$
\State $R_b^t\gets \operatorname{sg}(A_b^t)$
\For{$n=1,\ldots,N$}
    \State $s_n\gets \operatorname{cos\_sim}(\widehat A_{a\to b,n}^{\,t},R_{b,n}^{t})$
    \State $\delta_n\gets 1-s_n$
\EndFor
\State $d\gets \frac{1}{N}\sum_{n=1}^{N}\delta_n$
\State \Return $d$
\end{algorithmic}
\end{algorithm}

Equivalently, the transition distance used in \Cref{alg:realign_training} is
\begin{equation}
d(\widehat A_{a\to b}^{\,t},\operatorname{sg}(A_b^t))
=
\frac{1}{N}\sum_{n=1}^{N}
\left(
1-
\operatorname{cos\_sim}\left(\widehat A_{a\to b,n}^{\,t},\operatorname{sg}(A_{b,n}^{t})\right)
\right).
\label{eq:appendix_token_cosine_distance}
\end{equation}
The diversity term is computed directly from the hidden-state cache $\mathcal{H}$ and does not require an additional backbone forward pass.

\subsection{Transition Pair and Predictor Design}
\label{app:transition_pair_predictor_design}

Layer-pair sampling replaces the fixed layer choice used by direct self-alignment. Very short gaps make transition prediction nearly local, while very long gaps can make the target state too distant early in training. Unless otherwise stated, main runs use depth-centered gap sampling: draw the positive gap $g=b-a$ from a truncated normal distribution with $\mu=3$ and $\sigma=2$, draw the midpoint $m=(a+b)/2$ from a normal distribution centered at $L/2$, and reject invalid pairs. Gap-only, depth-center-only, and full-uniform alternatives are compared in \Cref{tab:pair_sampling_strategy_fid}. The predictor is shared across valid pairs through layer and timestep conditioning, so the auxiliary branch learns reusable cross-depth transitions instead of separate pair-specific adapters.

\subsection{Extended Related Work and Baseline Descriptions}
\label{app:baseline_descriptions}

In what follows, we summarize the main baseline methods used in our evaluation:

\begin{itemize}[leftmargin=*]
    \item \textbf{ADM}~\citep{dhariwal2021diffusion} builds upon U-Net-based diffusion models by introducing classifier-guided sampling, allowing fine-grained control over the trade-off between generation quality and diversity.
    \item \textbf{VDM++}~\citep{kingma2023understanding} proposes an adaptive noise schedule that adjusts dynamically during training, improving convergence and sample quality.
    \item \textbf{Simple diffusion}~\citep{hoogeboom2023simple} simplifies both the noise schedule and architectural components, enabling high-resolution image generation with improved computational efficiency.
    \item \textbf{CDM}~\citep{ho2022cascaded} introduces cascaded diffusion models that progressively refine images from low to high resolution using super-resolution stages, achieving better detail synthesis.
    \item \textbf{LDM}~\citep{rombach2022ldm} trains diffusion models in a compressed latent space learned by a VAE, drastically reducing training cost while maintaining image fidelity.
    \item \textbf{U-ViT}~\citep{bao2023all} combines ViT-based backbones with U-Net-style skip connections in the latent space, bridging the benefits of transformers and convolutional inductive biases.
    \item \textbf{DiffiT}~\citep{hatamizadeh2024diffit} enhances transformer-based diffusion models using time-aware multi-head self-attention, boosting sample efficiency and reducing training time.
    \item \textbf{MDTv2}~\citep{gao2023mdtv2} employs an asymmetric encoder-decoder transformer architecture with U-Net-inspired shortcuts in the encoder and dense skip connections in the decoder, improving generation quality and coherence.
    \item \textbf{DiT}~\citep{peebles2023scalable} proposes a pure transformer architecture for diffusion, using AdaLN-zero modules to stabilize training and scale to large model sizes efficiently.
    \item \textbf{SiT}~\citep{ma2024sit} investigates the link between training efficiency and flow-based perspectives by transitioning from discrete-time diffusion to continuous flow matching, showing improved sample quality and convergence rates.
    \item \textbf{REPA}~\citep{yu2024repa} improves DiT/SiT training by aligning hidden states to features from an external visual encoder such as DINOv2.
    \item \textbf{D-JEPA}~\citep{chen2024jepa} integrates Joint-Embedding Predictive Architectures into generative modeling by reframing masked image modeling as a generalized next-token prediction task, using diffusion or flow-matching losses to model per-token probability distributions in a continuous space.
    \item \textbf{VAR}~\citep{tian2024visual} redefines autoregressive image generation as a coarse-to-fine ``next-scale prediction'' process, diverging from standard raster-scan next-token prediction and allowing faster inference and scaling laws similar to large language models.
    \item \textbf{Dispersive Loss}~\citep{wang2025dispersive} regularizes internal features by encouraging diverse directions across hidden representations without relying on an external teacher.
    \item \textbf{SRA}~\citep{jiang2025sra} is an EMA-based method that enables diffusion transformers to enhance their own representation learning and generation quality by aligning latent outputs from earlier, noisier layers with those from later, cleaner layers, eliminating the need for external guidance models.
    \item \textbf{LayerSync}~\citep{haghighi2026layersync} directly aligns selected weak and strong internal layers in the diffusion transformer, providing a self-contained depth-alignment baseline.
\end{itemize}

\subsection{HumanML3D Setup}
\label{app:humanml3d_details}

\paragraph{Task.}
Given a sentence that describes a motion as a sequence of actions, the task is to generate a corresponding human motion. Each motion sequence consists of a series of human poses, where each pose is represented by 22 joints defined as 3D points in space.

\paragraph{Benchmark.}
We evaluate on HumanML3D~\citep{guo2022generating}, a standard text-to-motion benchmark built from motion-capture sequences and paired natural-language descriptions. To keep the comparison focused on the effect of internal alignment, all methods use the same motion representation, train/test split, text encoder, and evaluator as MDM~\citep{tevet2022mdm}. We therefore treat the HumanML3D preprocessing pipeline as fixed and vary only the internal training objective.

\paragraph{Implementation details.}
The motion backbone has eight transformer blocks. MDM and LayerSync are trained for 600K iterations, while LARA is evaluated at 300K and 400K training checkpoints. LayerSync applies its direct hidden-state alignment to the selected internal pair $(3,6)$. LARA uses the same backbone but replaces direct matching with the transition predictor described in \Cref{sec:method}.

Standard generation metrics follow the cited benchmark protocols. FID compares Gaussian approximations of real and generated Inception features,
\begin{equation}
\operatorname{FID}
=
\|\mu_r-\mu_g\|_2^2
+
\operatorname{Tr}\left(
\Sigma_r+\Sigma_g
-2(\Sigma_r\Sigma_g)^{1/2}
\right),
\end{equation}
where lower is better. IS is computed using the Inception classifier convention of \citet{salimans2016improved}. Precision and recall follow the improved generative-model support estimate of \citet{kynkaanniemi2019precision}. For HumanML3D, FID uses motion-evaluator embeddings and R-Precision reports whether the matching text appears in the top retrieved candidates under the standard HumanML3D/MDM evaluation protocol~\citep{guo2022generating,tevet2022mdm}.

For \Cref{fig:projection_layout_observation}, PCA maps are computed from channel-normalized token activations of frozen SiT-XL/2 hidden states for the same image over representative timesteps. We visualize the leading components as RGB maps to show how spatial structure is preserved while semantic content changes across depth and denoising time.

\textbf{Diagnostic readout.}
\Cref{tab:pair_sampling_ablation} shows that transition difficulty is highly non-uniform under full uniform sampling. Prediction error increases steadily with layer gap, from $0.025$ for the shortest quartile to $0.152$ for the longest quartile, indicating that very distant transitions are much noisier training targets for the auxiliary predictor. The source-layer grouping shows a complementary effect: middle layers are easier to predict from, while the earliest and latest source bins have larger errors. This supports the sampler used in the main experiments. Depth-centered gap sampling avoids spending too much probability mass on unstable long-range jumps, while centering the pair midpoint near the model's depth middle keeps supervision away from the most brittle boundary layers. The goal is not to remove hard transitions entirely, but to make the prediction loss informative without letting rare high-error pairs dominate the backbone update.

\section{Optional Inference and Layer-Skip Diagnostics}
\label{app:optional_inference_diagnostics}

\subsection{Optional Transition-Interface Inference}
\label{app:optional_transition_inference}

The main paper uses the standard full-backbone sampler. The transition interface is evaluated only as an appendix diagnostic: at each Euler step, a route policy $\pi(k)$ either keeps the full backbone or replaces a block range $(a,b)$ with the predictor followed by the suffix blocks. Setting $\pi(k)=\mathrm{full}$ for all $k$ recovers standard Euler exactly.

\begin{algorithm}[!htbp]
\caption{\textbf{Optional transition-interface Euler sampler.}}
\label{alg:realign_inference}
\small
\begin{algorithmic}[1]
\Require schedule $1=t_K>\cdots>t_0=0$; condition $y$; backbone $v_\theta$ with prefix/suffix maps $F_{\theta,1:a},F_{\theta,b:L}$; predictor $G_\psi$; route policy $\pi(k)\in\{\mathrm{full}\}\cup\{(a,b):a<b\}$.
\State Sample $z_K\sim\mathcal{N}(0,I)$.
\For{$k=K,\ldots,1$}
    \State $r_k\gets\pi(k)$.
    \If{$r_k=\mathrm{full}$}
        \State $\hat v_k\gets v_\theta(z_k,t_k,y)$.
    \Else
        \State $(a,b)\gets r_k$.
        \State $A_a^{t_k}\gets F_{\theta,1:a}(z_k,t_k,y)$.
        \State $\widehat A_b^{t_k}\gets G_\psi(A_a^{t_k};e_a,e_b,e_{t_k})$.
        \State $\hat v_k\gets F_{\theta,b:L}(\widehat A_b^{t_k},t_k,y)$.
    \EndIf
    \State $z_{k-1}\gets z_k+(t_{k-1}-t_k)\hat v_k$.
\EndFor
\State \Return $z_0$
\end{algorithmic}
\end{algorithm}

Each diagnostic must report the skip rate, chosen layer jumps, and whether the same route is used at all timesteps or only at selected Euler steps. The comparison uses the same initial noise, class labels, schedule, and checkpoint as the full-backbone sampler.

\subsection{Magnitude-Calibrated Layer-Skip Diagnostic}
\label{app:magnitude_layer_skip}

This diagnostic tests whether a predicted target-layer state can replace skipped blocks. Because the cosine transition loss mainly controls direction, we attach a small magnitude head $h_\omega$ that predicts the target token scale from $\operatorname{sg}(\widehat A_{a\to b}^{\,t})$. Its loss is fully detached, so it updates only $h_\omega$ and never backpropagates into $v_\theta$ or $G_\psi$. At evaluation, we normalize the predicted direction, apply the predicted scale, and feed the calibrated state to the suffix $F_{\theta,b:L}$. This route is appendix-only and is not used for the main generation results.

\section{Transition Predictor Configuration and Capacity}
\label{app:predictor_architecture_capacity}

We first specify how the transition predictor is attached to the SiT backbones, since the capacity sweep is only meaningful once the state geometry is fixed. LARA uses a compact ConvNet-style spatial predictor, implemented with ConvNeXt-style residual blocks~\citep{liu2022convnext}, on top of SiT hidden states. The latent tokens form a $16{\times}16$ grid at ImageNet $256{\times}256$, so lightweight local mixing can preserve the perceptual layout that matters for representation alignment, in the spirit of i-REPA~\citep{singh2025irepa}. The predictor takes a source-layer activation, injects source-layer, target-layer, and timestep conditioning, and returns an activation in the target-layer coordinate system. The target branch is not processed by the predictor; it is used as a stopped-gradient reference for the distance in \Cref{alg:lara_transition_distance}.
