\section{Method}

\subsection{Preliminaries}

We formulate our method within the stochastic-interpolant view adopted by Scalable Interpolant Transformers (SiTs)~\citep{ma2024sit}, which provides a common language for flow-based and diffusion-based generative models. In this view, generation is described as learning a continuous transport process between a simple base distribution and the data distribution. Given a clean sample $x_0$ and a Gaussian noise sample $\epsilon$, an interpolant constructs an intermediate state
\begin{equation}
x_t = \alpha_t x_0 + \sigma_t \epsilon ,
\label{eq:stochastic_interpolant}
\end{equation}
where $t$ denotes time and $\alpha_t,\sigma_t$ determine how much data and noise contribute along the path. For small $t$, the state is close to the data endpoint under common schedules; for large $t$, it approaches the noise endpoint. Sampling then amounts to starting from noise and following a learned vector field back toward the data distribution.

The instantaneous direction of this path is obtained by differentiating the interpolant with respect to time, yielding the target velocity
\begin{equation}
u_t(x_0,\epsilon)
=
\dot{\alpha}_t x_0+\dot{\sigma}_t\epsilon .
\label{eq:interpolant_velocity}
\end{equation}
Since $x_0$ and $\epsilon$ are not available at generation time, a neural network is trained to predict this velocity from the current latent $x_t$, timestep $t$, and optional condition $c$. We denote the SiT backbone by $f_\zeta$ and its velocity head by $v_\zeta$. The standard velocity-matching objective is
\begin{equation}
\mathcal{L}_{\mathrm{vel}}
=
\mathbb{E}_{x_0,\epsilon,t}
\left[
\left\|
v_\zeta(x_t,t,c)
-
(\dot{\alpha}_t x_0+\dot{\sigma}_t\epsilon)
\right\|_2^2
\right].
\label{eq:flow_matching_loss}
\end{equation}
After training, new samples are generated by drawing an initial noise sample and integrating the learned velocity field in the reverse direction.

\subsection{Overview}

We propose \emph{ReSync}, a self-supervised representation learning objective for self-aligned transition consistency in Diffusion Transformers. The main idea is simple: instead of only using intermediate hidden states as isolated features, we use their evolution across depth as supervision. Given a hidden state from an earlier layer, a lightweight predictor is trained to estimate the corresponding hidden state at a later layer. This encourages the backbone to learn representations whose cross-layer evolution is predictable and useful for generation.

Let a DiT backbone produce intermediate hidden states $Z_0,\ldots,Z_L$, where $Z_l\in\mathbb{R}^{B\times P\times D}$ denotes the representation after the $l$-th transformer block. The model is still trained with the original diffusion or flow-matching objective, denoted as $\mathcal{L}_{\mathrm{gen}}$. ReSync adds an auxiliary transition-consistency objective for predicting normalized target-layer representations.



\subsection{ReSync Transition Predictor}

For each intermediate layer, we apply tokenwise $\ell_2$ normalization to the hidden state before computing the ReSync objective. This removes the effect of varying activation scales across layers and timesteps, allowing the predictor to focus on the representation direction. We denote the normalized hidden state at layer $l$ as $A_l$.

Given a sampled pair of layers $a<b$, the transition predictor takes the normalized source representation $A_a$, the layer-pair condition $(a,b)$, and the timestep condition as input. It predicts the normalized representation of the target layer:
\begin{equation}
\hat A_{a\rightarrow b}
=
P_\theta(A_a,a,b,t).
\label{eq:transition_predictor}
\end{equation}

The predictor is a lightweight ConvNeXt-style network conditioned on learned layer embeddings and the DiT timestep embedding. Rather than changing the DiT architecture, it is used as an auxiliary module during training to provide cross-depth representation supervision.

\subsection{Auxiliary ReSync Loss}

For a sampled pair of layers $a<b$, ReSync trains the transition predictor to recover the normalized target representation from the source representation. We define the direct transition loss as
\begin{equation}
\mathcal{L}_{\mathrm{direct}}
=
\mathbb{E}_{a<b}
\left[
d\left(
\hat A_{a\rightarrow b},
\mathrm{sg}(A_b)
\right)
\right],
\label{eq:direct_transition_loss}
\end{equation}
where $d(\cdot,\cdot)$ denotes a tokenwise representation distance, implemented as cosine distance over normalized features.

We also regularize the predictor with bootstrap transition consistency. For a triplet
$a<b<c$, there are two ways to obtain a prediction for layer $c$ from layer $a$.
The first way is a composed transition: we first predict the intermediate
representation at layer $b$,
\[
\hat A_{a\rightarrow b}=P_\theta(A_a,a,b,t),
\]
and then feed this predicted representation back into the same predictor to
estimate layer $c$,
\[
\hat A_{a\rightarrow b\rightarrow c}
=
P_\theta(\hat A_{a\rightarrow b},b,c,t).
\]
The second way is a direct long transition from $a$ to $c$, produced by an EMA
copy of the predictor. We encourage these two predictions to agree:
\begin{equation}
\mathcal{L}_{\mathrm{boot}}
=
\mathbb{E}_{a<b<c}
\left[
d\left(
\hat A_{a\rightarrow b\rightarrow c},
\mathrm{sg}(\hat A^{\mathrm{ema}}_{a\rightarrow c})
\right)
\right].
\label{eq:bootstrap_transition_loss}
\end{equation}

The auxiliary ReSync objective is
\begin{equation}
\mathcal{L}_{\mathrm{ReSync}}
=
\mathcal{L}_{\mathrm{direct}}
+
\alpha\mathcal{L}_{\mathrm{boot}}.
\label{eq:resync_loss}
\end{equation}

The direct term teaches the predictor to recover later-layer representations, while the bootstrap term encourages the learned cross-depth transition to be consistent when composed over multiple shorter steps. All targets are stop-gradient, and the EMA predictor is used only to provide stable bootstrap targets.

\subsection{Training Objective}

The final training objective is the original generative loss plus the ReSync auxiliary loss:
\begin{equation}
\mathcal{L}
=
\mathcal{L}_{\mathrm{gen}}
+
\lambda_{\mathrm{ReSync}}\mathcal{L}_{\mathrm{ReSync}}.
\label{eq:total_loss}
\end{equation}

Gradients from the auxiliary loss are allowed to flow through the source hidden state, so ReSync acts as a self-supervised representation learning objective for the DiT backbone. The method therefore obtains representation guidance entirely from the model's own hidden-state trajectory, without requiring an external representation task or a pretrained visual encoder.
