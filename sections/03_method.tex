\section{Method}
\label{sec:method}

\subsection{Preliminaries}

We adopt the stochastic-interpolant formulation used by SiT~\citep{ma2024sit}, which provides a common view of flow-based and diffusion-based generative models. Given a data latent $x_0\sim p_{\mathrm{data}}$, Gaussian noise $\epsilon\sim\mathcal{N}(0,I)$, and timestep $t\in[0,1]$, the interpolation path is
\begin{equation}
x_t=\alpha_t x_0+\sigma_t\epsilon ,
\label{eq:stochastic_interpolant}
\end{equation}
where $\alpha_t$ and $\sigma_t$ control the mixture of data and noise. The schedule is chosen so that the path connects clean data to a tractable noise distribution, and generation follows the learned dynamics in the reverse direction. The instantaneous velocity of this path is
\[
u_t=\dot{\alpha}_t x_0+\dot{\sigma}_t\epsilon .
\]
A diffusion transformer $v_\zeta$ receives the noisy latent $x_t$, timestep $t$, and optional condition $c$, and is trained to predict this velocity by minimizing
\begin{equation}
\mathcal{L}_{\mathrm{FM}}
=
\mathbb{E}_{x_0,\epsilon,t,c}\left[
\left\|v_\zeta(x_t,t,c)-u_t\right\|_2^2
\right].
\label{eq:flow_matching_loss}
\end{equation}

\subsection{From Projection Heads to Transition Prediction}

\paragraph{Representation hierarchy.}
Diffusion transformers do not expose a uniform stack of interchangeable hidden states. Prior analyses show that intermediate layers differ in semantic quality, with useful features often peaking before the final decoder-like blocks~\citep{mukhopadhyay2024,kim2025,stracke2025,haghighi2026layersync}. We observe the same pattern in a pretrained SiT-XL/2: activation scale drifts across depth, while CKA to DINOv2 varies substantially by layer (\Cref{fig:sit_xl_layer_statistics}). The first effect reflects a statistical mismatch between raw layer states, consistent with signal-propagation and Transformer normalization analyses~\citep{glorot2010understanding,he2015delving,schoenholz2017deep,liu2020understandingtransformers}; the second reflects a representation hierarchy.

\paragraph{Projection as interface.}
LayerSync exploits the hierarchy by aligning weaker layers to richer internal ones~\citep{haghighi2026layersync}, but direct alignment also assumes that source and target activations are comparable in raw coordinates. This makes layer choice a coupled problem: a poor source layer may be semantically weak, statistically mismatched, or both. REPA and SRA avoid raw matching by inserting projection heads before matching external or internal targets~\citep{yu2024repa,jiang2025sra}. \paragraph{Signal filtering.}
The weak-layer rows are the key comparison. If projection improves them, the source is not simply uninformative; its useful signal is poorly exposed in raw coordinates. In a residual transformer, $A_b^t=A_a^t+\Delta_{a:b}^t$, so a projector satisfying $P_{a\to b}(A_a^t)\approx A_b^t$ learns an implicit internal mapping whose residual $P_{a\to b}(A_a^t)-A_a^t$ approximates knowledge added by later blocks. The interface can therefore distill information between representation spaces and filter the auxiliary signal before it updates the flow-matching model. We test the filtering view in the appendix with gradient cosine between $\mathcal{L}_{\mathrm{FM}}$ and direct versus projected alignment losses.

\paragraph{Recoverable layout.}
The recoverable signal may include spatial structure, not only global semantics. iREPA shows that preserving spatial structure in REPA targets can matter beyond global classification strength~\citep{singh2025irepa}. We therefore use PCA and stable rank in \Cref{fig:projection_layout_observation} as a compact qualitative check: low stable rank means a few principal directions explain much of the variation, making PCA maps informative about coarse layout~\citep{daneshmand2020batchnormrankcollapse,feng2022rankdiminishing}. We treat layout recovery as a hypothesis rather than a primary claim.

\subsection{ReSync: Self-Aligned Transition Consistency}

ReSync is designed to use the backbone's depth trajectory as a source of self-supervision. Instead of selecting a small set of teacher layers or training a separate adapter for each pair, it samples transitions across the valid layer range and trains one conditional predictor between internal representation spaces. Let $A_\ell^t\in\mathbb{R}^{N\times C}$ denote the token activation after block $\ell$ when the diffusion transformer processes $(x_t,t,c)$. For a source layer $a$ and a later target layer $b$, ReSync predicts the target representation from the source representation at the same denoising timestep:
\begin{equation}
T_{a\to b}^t(A)\equiv T_\psi(A,a,b,t),
\qquad
\hat A_{b\mid a}^t=T_{a\to b}^t(A_a^t).
\label{eq:resync_predictor}
\end{equation}
The predictor is shared across layer pairs and conditioned on $(a,b,t)$. This lets signals that appear at different depths train the same transition function, while the conditioning preserves the source and target coordinates of each signal. In this sense, ReSync learns an implicit internal representation mapping: in a residual stack, predicting $A_b^t$ from $A_a^t$ amounts to estimating the accumulated transformation from layer $a$ to layer $b$ in the target layer's coordinates. Unless stated otherwise, $\mathbb{E}$ averages over data, noise, timesteps, and sampled layer indices; $d(\cdot,\cdot)$ is tokenwise cosine distance; and $\mathrm{sg}(\cdot)$ denotes stop-gradient. The self-alignment loss is
\begin{equation}
\mathcal{L}_{\mathrm{pred}}
=
\mathbb{E}\left[
d\left(\hat A_{b\mid a}^t,\mathrm{sg}(A_b^t)\right)
\right].
\label{eq:align_loss}
\end{equation}
The target activation is detached, so supervision flows through the transition interface while avoiding symmetric collapse between source and target states.

Prediction alone can still behave like a set of unrelated adapters. To avoid this, ReSync imposes path consistency. For a triplet $a<b<c$, the direct transition should agree with the composition of two shorter transitions:
\begin{equation}
\hat A_{c\mid a}^t
\approx
\hat A_{c\mid b\mid a}^t .
\label{eq:transition_composition}
\end{equation}
Here $\hat A_{c\mid a}^t$ is produced by one application of $T_\psi$, while $\hat A_{c\mid b\mid a}^t$ is obtained by applying the transitions $a\to b$ and $b\to c$. We form the composed target with an EMA copy $\bar T$ of the predictor, writing $\bar T_{p\to q}^t(A)\equiv\bar T(A,p,q,t)$ and following the common use of weight-averaged consistency targets~\citep{tarvainen2017meanteacher}:
\begin{equation}
\tilde A_a^t=\mathrm{sg}(A_a^t),
\qquad
\bar A_{b\mid a}^t=\bar T_{a\to b}^{\,t}(\tilde A_a^t),
\qquad
\bar A_{c\mid b\mid a}^t
=
\bar T_{b\to c}^{\,t}(\bar A_{b\mid a}^t).
\label{eq:composed_target}
\end{equation}
The online predictor is then trained against this composed target:
\begin{equation}
\mathcal{L}_{\mathrm{cons}}
=
\mathbb{E}\left[
d\left(T_{a\to c}^t(\tilde A_a^t),\mathrm{sg}(\bar A_{c\mid b\mid a}^t)\right)
\right].
\label{eq:consistency_loss}
\end{equation}
This constraint is the main distinction from pairwise self-alignment. The same conditional predictor must explain local transitions and longer-range transitions sampled throughout the model, making ReSync a consistent internal representation-transition objective rather than a collection of independent matching losses.

\subsection{Transition Pair and Predictor Design}

The choice of transition pairs plays a role analogous to layer selection in direct self-alignment. ReSync uses a shared predictor, so we avoid designing a separate head for each source--target pair. Instead, we sample pairs from a bounded range of layer gaps. Very short gaps make the prediction task nearly trivial, while very long gaps can make the target representation too distant to provide stable supervision early in training. In practice, we use a mixture of short and medium gaps and ablate this sampling strategy in the experiments.

We also avoid relying on the final decoder-like blocks as the dominant targets. Prior analyses of diffusion representations indicate that semantic quality is heterogeneous across depth and that the last blocks become increasingly tied to output reconstruction~\citep{mukhopadhyay2024,kim2025,stracke2025,haghighi2026layersync}. We therefore treat the valid source range, target range, and gap distribution as predictor-design choices rather than fixed assumptions. Finally, the predictor itself is kept lightweight and shared across pairs through layer and timestep conditioning. This design keeps the method self-contained while testing whether one compact module can capture reusable internal transitions instead of memorizing pair-specific alignments.

\subsection{Objective}

The main training objective combines the prediction and compositional consistency losses above with the base flow-matching objective. For exploratory skipped-depth ablations, we also consider an optional output-level distillation term. Given a predicted activation, we pass it through the remaining backbone,
\begin{equation}
\hat y_{\mathrm{skip}}
=
F_{\zeta,b:L}(\hat A_b^t,t,c),
\qquad
\hat y_{\mathrm{full}}
=
F_{\zeta,b:L}(A_b^t,t,c),
\label{eq:skip_output}
\end{equation}
and match the resulting output to the full computation:
\begin{equation}
\mathcal{L}_{\mathrm{out}}
=
\left\|
\hat y_{\mathrm{skip}}
-
\mathrm{sg}(\hat y_{\mathrm{full}})
\right\|_2^2 .
\label{eq:output_distillation_loss}
\end{equation}
This term is not required for the main method. We use it only as an ablation to make the predictor more robust to its own errors and to encourage predicted activations to preserve information useful for the final velocity prediction.

We collect the auxiliary terms into a single predictor objective,
\begin{equation}
\mathcal{L}_{\mathrm{predictor}}
=
\mathcal{L}_{\mathrm{pred}}
+
\beta_{\mathrm{cons}}\mathcal{L}_{\mathrm{cons}}
+
\beta_{\mathrm{out}}\mathcal{L}_{\mathrm{out}} .
\label{eq:predictor_loss}
\end{equation}
The resulting training objective is
\begin{equation}
\mathcal{L}
=
\mathcal{L}_{\mathrm{FM}}
+
\lambda_{\mathrm{pred}}\mathcal{L}_{\mathrm{predictor}} .
\label{eq:total_loss}
\end{equation}
In all main experiments, $\beta_{\mathrm{out}}=0$, so ReSync is purely a training-time representation regularizer and adds no inference-time cost.
