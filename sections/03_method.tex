\section{Method}
\label{sec:method}

\subsection{Diffusion Transformer Training}

We adopt the stochastic-interpolant formulation used by SiT~\citep{ma2024sit}, which provides a common view of flow-based and diffusion-based generative models. Given a data latent $x_0\sim p_{\mathrm{data}}$, Gaussian noise $\epsilon\sim\mathcal{N}(0,I)$, and timestep $t\in[0,1]$, the interpolation path is
\begin{equation}
x_t=\alpha_t x_0+\sigma_t\epsilon ,
\label{eq:stochastic_interpolant}
\end{equation}
where $\alpha_t$ and $\sigma_t$ control the mixture of data and noise. The schedule is chosen so that the path connects clean data to a tractable noise distribution, and generation follows the learned dynamics in the reverse direction. The instantaneous velocity of this path is
\[
u_t=\dot{\alpha}_t x_0+\dot{\sigma}_t\epsilon .
\]
A diffusion transformer $v_\zeta$ receives the noisy latent $x_t$, timestep $t$, and optional condition $c$, and is trained to predict this velocity by minimizing
\begin{equation}
\mathcal{L}_{\mathrm{FM}}
=
\mathbb{E}_{x_0,\epsilon,t,c}\left[
\left\|v_\zeta(x_t,t,c)-u_t\right\|_2^2
\right].
\label{eq:flow_matching_loss}
\end{equation}

\paragraph{Projection as interface.}
LayerSync exploits the hierarchy by aligning weaker layers to richer internal ones~\citep{haghighi2026layersync}. Direct alignment, however, still assumes that source and target activations are comparable in raw coordinates. REPA and SRA avoid raw matching by inserting projection heads before matching external or internal targets~\citep{yu2024repa,jiang2025sra}. \Cref{tab:layersync_projector_probe} isolates this interface choice.

\paragraph{Signal filtering.}
The weak-layer rows are the key comparison. If a small interface improves them, the useful signal was already present but poorly exposed in raw coordinates. The uniform and layer-aware rows then ask whether the same idea remains useful when supervision is spread across many pairs. With residual blocks, $A_b^t=A_a^t+\Delta_{a:b}^t$, so a map satisfying $P_{a\to b}(A_a^t)\approx A_b^t$ recovers part of the accumulated depth update in the target layer's coordinates. This is the step from direct matching to transition prediction.

\paragraph{Recoverable layout.}
The recoverable signal can include spatial structure as well as global semantics. iREPA shows that spatial structure in alignment targets can matter in addition to classification strength~\citep{singh2025irepa}. We use PCA and stable rank in \Cref{fig:projection_layout_observation} as a compact check of whether projected states retain organized spatial variation~\citep{daneshmand2020batchnormrankcollapse,feng2022rankdiminishing}.

\subsection{Layer-Aware Cross-Depth Transition Prediction}

ReSync turns internal layer alignment into cross-depth transition prediction. It samples source--target layers $a<b$ and uses one shared predictor for all valid pairs. Let $A_\ell^t\in\mathbb{R}^{N\times C}$ be the token activation after block $\ell$ at denoising timestep $t$. The layer-aware transition predictor is
\begin{equation}
\hat A_{a\to b}^t
=
g_\psi(A_a^t; a,b,t),
\qquad a<b .
\label{eq:resync_predictor}
\end{equation}
The same function $g_\psi$ is reused across depth transitions, while $(a,b,t)$ specifies the source, target, and denoising stage. This makes internal alignment flexible across layer pairs without a separate adapter for each pair. 

The basic ReSync loss asks the predicted target-layer state to match the actual later-layer state:
\begin{equation}
\mathcal{L}_{\mathrm{pred}}
=
\mathbb{E}_{a<b,\,t}\Big[\,
\underbrace{\,d\big(\hat A_{a\to b}^{\,t},\; \mathrm{sg}(A_b^t)\,\big)\,}_{\text{predicted layer-}b\text{ state vs.\ actual layer-}b\text{ state}}
\Big],
\label{eq:align_loss}
\end{equation}
where $d$ is tokenwise cosine distance and $\mathrm{sg}$ denotes stop-gradient. The detached target directs supervision through the transition interface and source-side representation.

We add a lightweight path-consistency term so that short and long transitions share the same interface. Using an EMA copy $\bar g_\psi$ of the predictor~\citep{tarvainen2017meanteacher}, we form the composed target in two steps,
\begin{equation}
\hat A_{a\to b}^{\,t,\,\bar\psi}
= \bar g_\psi\!\big(\mathrm{sg}(A_a^t);\, a,b,t\big),
\qquad
\hat A_{a\to c}^{\,t,\,\mathrm{via}\,b}
= \bar g_\psi\!\big(\hat A_{a\to b}^{\,t,\,\bar\psi};\, b,c,t\big),
\label{eq:composed_target}
\end{equation}
and require the direct prediction $\hat A_{a\to c}^{\,t}=g_\psi(\mathrm{sg}(A_a^t);a,c,t)$ to agree with it:
\begin{equation}
\mathcal{L}_{\mathrm{cons}}
=
\mathbb{E}_{a<b<c,\,t}\Big[\,
\underbrace{\,d\big(\hat A_{a\to c}^{\,t},\; \mathrm{sg}(\hat A_{a\to c}^{\,t,\,\mathrm{via}\,b})\,\big)\,}_{\text{direct prediction vs.\ composed (EMA) prediction}}
\Big].
\label{eq:consistency_loss}
\end{equation}

This keeps the shared interface coordinated across short and long transitions. The effect is simple: layer supervision becomes a reusable transition signal across many layer pairs. Implementation choices for layer ranges, gap sampling, and predictor capacity are given in \Cref{app:transition_pair_predictor_design}.

For a small subset of each minibatch, ReSync feeds the predicted target state to the remaining transformer blocks and trains it with the ordinary flow-matching target. Let $F_{\zeta,b:L}$ denote the suffix of the diffusion transformer from block $b$ to the output. The velocity prediction remains a single quantity:
\begin{equation}
\hat v
=
\begin{cases}
F_{\zeta,1:L}(x_t,t,c), & \text{standard example},\\
F_{\zeta,b:L}(\hat A_{a\to b}^t,t,c), & \text{FM-path example}.
\end{cases}
\label{eq:fm_path_forward}
\end{equation}
For FM-path examples, the second line is used. The same loss $\|\hat v-u_t\|_2^2$ is applied in both cases. This keeps transition prediction connected to the generative objective without adding a separate output-matching loss.

To keep layers from collapsing into overly similar states, we add a lightweight diversity term inspired by DiverseDiT~\citep{yang2026diversedit}. For randomly sampled layer pairs $(i,j)$, independent of the transition pair used in $\mathcal{L}_{\mathrm{pred}}$, we use
\begin{equation}
\mathcal{L}_{\mathrm{div}}
=
\mathbb{E}_{i<j,t}
\left[
\operatorname{cos}^2(A_i^t,A_j^t)
\right].
\label{eq:diversity_loss}
\end{equation}
It penalizes high direct similarity between sampled layer activations while leaving the transition targets unchanged; no-diversity ablations isolate its effect.

\paragraph{Optimization details.}
The stop-gradient choices keep the roles of the terms clear. $\mathcal{L}_{\mathrm{pred}}$ updates the predictor and source-side backbone activations against a detached target state. $\mathcal{L}_{\mathrm{cons}}$ updates the online predictor against the detached EMA-composed target. $\mathcal{L}_{\mathrm{div}}$ is applied only to backbone activations.

\subsection{Full Objective}

ReSync is added to the flow-matching objective as an internal regularizer. The representation objective combines transition prediction, path consistency, and weak layer diversity:
\begin{equation}
\mathcal{L}_{\mathrm{ReSync}}
=
\mathcal{L}_{\mathrm{pred}}
+
\beta_{\mathrm{cons}}\mathcal{L}_{\mathrm{cons}}
+
\beta_{\mathrm{div}}\mathcal{L}_{\mathrm{div}} .
\label{eq:resync_loss}
\end{equation}
The final training objective is
\begin{equation}
\mathcal{L}
=
\mathcal{L}_{\mathrm{FM}}
+
\lambda_{\mathrm{sync}}\mathcal{L}_{\mathrm{ReSync}} .
\label{eq:total_loss}
\end{equation}
The predictor is used only during training and is discarded during standard sampling, so ReSync adds no inference-time cost. Training and inference procedures, architecture, parameter overhead, and loss weights are reported in \Cref{app:resync_algorithm_compute,app:resync_hyperparameters}.
