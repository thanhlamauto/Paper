\section{Method}
\label{sec:method}

\subsection{Preliminaries}

We adopt the stochastic-interpolant formulation used by SiT~\citep{ma2024sit}, which provides a common view of flow-based and diffusion-based generative models. Given a data latent $x_0\sim p_{\mathrm{data}}$, Gaussian noise $\epsilon\sim\mathcal{N}(0,I)$, and timestep $t\in[0,1]$, the interpolation path is
\begin{equation}
x_t=\alpha_t x_0+\sigma_t\epsilon ,
\label{eq:stochastic_interpolant}
\end{equation}
where $\alpha_t$ and $\sigma_t$ control the mixture of data and noise. The schedule is chosen so that the path connects clean data to a tractable noise distribution, and generation follows the learned dynamics in the reverse direction. The instantaneous velocity of this path is
\[
u_t=\dot{\alpha}_t x_0+\dot{\sigma}_t\epsilon .
\]
A diffusion transformer $v_\zeta$ receives the noisy latent $x_t$, timestep $t$, and optional condition $c$, and is trained to predict this velocity by minimizing
\begin{equation}
\mathcal{L}_{\mathrm{FM}}
=
\mathbb{E}_{x_0,\epsilon,t,c}\left[
\left\|v_\zeta(x_t,t,c)-u_t\right\|_2^2
\right].
\label{eq:flow_matching_loss}
\end{equation}

\subsection{From Projection Interfaces to Self-Alignment}

\paragraph{Representation hierarchy.}
Diffusion transformers do not expose a uniform stack of interchangeable hidden states. Prior analyses show that intermediate layers differ in semantic quality, with useful features often peaking before the final decoder-like blocks~\citep{mukhopadhyay2024,kim2025,stracke2025,haghighi2026layersync}. We observe the same pattern in a pretrained SiT-XL/2: activation scale drifts across depth, while CKA to DINOv2 varies substantially by layer (\Cref{fig:sit_xl_layer_statistics}). The first effect reflects a statistical mismatch between raw layer states, consistent with signal-propagation and Transformer normalization analyses~\citep{glorot2010understanding,he2015delving,schoenholz2017deep,liu2020understandingtransformers}; the second reflects a representation hierarchy.

\paragraph{Projection as interface.}
LayerSync exploits the hierarchy by aligning weaker layers to richer internal ones~\citep{haghighi2026layersync}, but direct alignment also assumes that source and target activations are comparable in raw coordinates. This makes layer choice a coupled problem: a poor source layer may be semantically weak, statistically mismatched, or both. REPA and SRA avoid raw matching by inserting projection heads before matching external or internal targets~\citep{yu2024repa,jiang2025sra}. We isolate this design choice in \Cref{tab:layersync_projector_probe}.

\paragraph{Signal filtering.}
The weak-layer rows are the key comparison. If projection improves them, the source is not simply uninformative; its useful signal is poorly exposed in raw coordinates. In a transformer architecture with residual blocks, $A_b^t=A_a^t+\Delta_{a:b}^t$, so a projector satisfying $P_{a\to b}(A_a^t)\approx A_b^t$ learns the recoverable part of this accumulated update in the target layer's coordinates. The interface can therefore extract useful supervision from a source activation and filter layer-specific mismatch before the auxiliary loss updates the flow-matching model.

\paragraph{Recoverable layout.}
The recoverable signal may include spatial structure, not only global semantics. iREPA shows that preserving spatial structure in REPA targets can matter beyond global classification strength~\citep{singh2025irepa}. We therefore use PCA and stable rank in \Cref{fig:projection_layout_observation} as a compact qualitative check: low stable rank means a few principal directions explain much of the variation, making PCA maps informative about coarse layout~\citep{daneshmand2020batchnormrankcollapse,feng2022rankdiminishing}. We treat layout recovery as a hypothesis rather than a primary claim.

\subsection{ReSync: Layer-Aware Self-Alignment}

ReSync turns the projection-interface view into a shared self-alignment objective. Instead of fixing one teacher layer pair, it samples source--target layers $a<b$ and uses one predictor for all of them. Let $A_\ell^t\in\mathbb{R}^{N\times C}$ be the token activation after block $\ell$ at denoising timestep $t$. We write the shared predictor as
\begin{equation}
\hat A_{a\to b}^t
=
g_\psi(A_a^t; a,b,t),
\qquad a<b .
\label{eq:resync_predictor}
\end{equation}
The notation exposes the intended role: the same function $g_\psi$ is reused for many depth transitions, while $(a,b,t)$ tells it where the transition starts, where it ends, and at which denoising stage it is used. This makes the interface flexible across layer pairs without turning each pair into its own adapter. It also shifts the auxiliary task from matching endpoints to modeling the relation between them, encouraging the predictor to capture what is shared across internal transitions rather than only memorizing a few selected matches.

The basic ReSync loss asks the predicted target-layer state to match the actual later-layer state:
\begin{equation}
\mathcal{L}_{\mathrm{pred}}
=
\mathbb{E}_{a<b}
\left[
d\left(\hat A_{a\to b}^t,\mathrm{sg}(A_b^t)\right)
\right],
\label{eq:align_loss}
\end{equation}
where $d$ is tokenwise cosine distance and $\mathrm{sg}$ denotes stop-gradient. The target is detached, so supervision flows through the transition interface and source-side representation, not by pulling both states symmetrically together.

A shared predictor should behave consistently when a long transition can be viewed as two shorter ones. For $a<b<c$, we compare the direct transition $a\to c$ with the composed path $a\to b\to c$:
\begin{equation}
g_\psi(\mathrm{sg}(A_a^t);a,c,t)
\approx
\hat A_{a\to c}^{t,\mathrm{via}\ b}.
\label{eq:transition_composition}
\end{equation}
The composed target is formed with an EMA copy $\bar g_\psi$ of the predictor~\citep{tarvainen2017meanteacher}:
\begin{equation}
\hat A_{a\to c}^{t,\mathrm{via}\ b}
=
\bar g_\psi\left(
    \bar g_\psi(\mathrm{sg}(A_a^t);a,b,t);
    b,c,t
\right).
\label{eq:composed_target}
\end{equation}
The consistency loss is then
\begin{equation}
\mathcal{L}_{\mathrm{cons}}
=
\mathbb{E}_{a<b<c}
\left[
d\left(
g_\psi(\mathrm{sg}(A_a^t);a,c,t),
\mathrm{sg}\left(\hat A_{a\to c}^{t,\mathrm{via}\ b}\right)
\right)
\right].
\label{eq:consistency_loss}
\end{equation}
This regularizer keeps the shared interface coordinated across short and long transitions. The effect is deliberately simple: layer supervision becomes a small, reusable transition system rather than a list of fixed pairwise matches. Implementation choices for layer ranges, gap sampling, and predictor capacity are given in \Cref{app:transition_pair_predictor_design}.

Hidden-state prediction is useful only if the predicted state can still support the generative task. Rather than distilling the output of the unmodified backbone, ReSync can route a small fraction of the minibatch through the predictor inside the ordinary flow-matching loss. Let $F_{\zeta,b:L}$ denote the suffix of the diffusion transformer from block $b$ to the output. For a routed example, the velocity prediction is
\begin{equation}
\hat v_{\mathrm{route}}
=
F_{\zeta,b:L}(\hat A_{a\to b}^t,t,c).
\label{eq:routed_fm_forward}
\end{equation}
It is compared to the same velocity target $u_t$ used in \Cref{eq:flow_matching_loss}. Thus routing does not introduce a new objective; it only changes the forward path for part of the minibatch estimate of $\mathcal{L}_{\mathrm{FM}}$.

To preserve functional separation between blocks, we add a lightweight diversity term inspired by DiverseDiT~\citep{yang2026diversedit}. For randomly sampled layer pairs $(i,j)$, independent of the transition pair used in $\mathcal{L}_{\mathrm{pred}}$, we use
\begin{equation}
\mathcal{L}_{\mathrm{div}}
=
\mathbb{E}_{i<j,t}
\left[
\operatorname{cos}^2(A_i^t,A_j^t)
\right].
\label{eq:diversity_loss}
\end{equation}
It penalizes high direct similarity between sampled layer activations while leaving the transition targets unchanged; no-diversity ablations isolate its effect.

\paragraph{Gradient routing.}
We route gradients according to the role of each term. $\mathcal{L}_{\mathrm{pred}}$ updates the predictor and source-side backbone activations, while the target activation is stopped. $\mathcal{L}_{\mathrm{cons}}$ updates the online predictor against the detached EMA-composed target. Routed examples, when used, update the routed prefix, predictor, and active suffix through $\mathcal{L}_{\mathrm{FM}}$. $\mathcal{L}_{\mathrm{div}}$ is applied only to backbone activations and does not update the transition predictor.

\subsection{Objective}

ReSync is added to the flow-matching objective as an internal regularizer. The representation objective combines transition prediction, path consistency, and weak layer diversity:
\begin{equation}
\mathcal{L}_{\mathrm{ReSync}}
=
\mathcal{L}_{\mathrm{pred}}
+
\beta_{\mathrm{cons}}\mathcal{L}_{\mathrm{cons}}
+
\beta_{\mathrm{div}}\mathcal{L}_{\mathrm{div}} .
\label{eq:resync_loss}
\end{equation}
The final training objective is
\begin{equation}
\mathcal{L}
=
\mathcal{L}_{\mathrm{FM}}
+
\lambda_{\mathrm{sync}}\mathcal{L}_{\mathrm{ReSync}} .
\label{eq:total_loss}
\end{equation}
The predictor is used only during training and is discarded during standard sampling, so ReSync adds no inference-time cost. Training and inference procedures, architecture, parameter overhead, and loss weights are reported in \Cref{app:resync_algorithm_compute,app:resync_hyperparameters}.
